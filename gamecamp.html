Top-Down Instruction:
A top-down teaching style focuses on providing students a large view of a subject, immersing them in the big picture without explaining the components that make up the subject. For example, with coding, a top-down approach would begin by immersing students in game coding immediately. Students would not be taught the remedial intricacies first, instead they would be plunged into the totality of learning to use code to build games and then gradually learn the building blocks that make up the JavaScript language.
Project-Based learning.
Learn coding by building game projects. Learn what they need to know as they need to know it.

Learning Objectives

How are games made?

Cover the 3 displines of game development:
Art
the visuals of the game world & game objects.
Design
the layout of the game world & game objects.
Code
the rules & logic that make the game play.
End of Camp Objective:

After 1 week, campers should know the tech well enough to continue to learn on their own via online resources. We'll build 2-3 games completely from the ground-up, such that campers should understand both the Plan-Design-Implement-Test process & the basic architecture for making games... and most importantly, to have fun while learning STEM!

Mathematical Modeling

Campers learn how to use mathematical expressions to model their game's actions. This help provide them meaning and context as why math is useful.
Campers learn why math is important and how to use it effectively to model actions within their games such as defining character movements, character attacks, gravity, and more. Everything in their game world depends on math! None ofwhich is possible without applying math.
